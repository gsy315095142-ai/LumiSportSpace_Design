💡 数据加载策略

onMounted：初始化导航栏 + 获取用户资料 + 加载钱包余额
onShow：每次页面可见时重新刷新钱包和用户资料，确保余额与昵称为最新状态

👤个人资料 · MineProfileCard

展示用户昵称和角色，支持点击「修改昵称」弹窗编辑：

  • 昵称展示 — 来自 userStore.userInfo.nickname，为空时显示「未设置昵称」
  • 角色标识 — admin 显示「管理员」，user 显示「用户」
  • 修改昵称弹窗 — 输入框预填当前昵称（trimmed），最大长度 100 字符，允许清空
  • 保存逻辑 — 调用 PUT /api/auth/profile，成功 Toast「已保存」并关闭弹窗，失败 Toast 错误信息

💡 交互细节

点击蒙层（mask）可关闭弹窗（等同于取消）。保存中按钮处于 disabled 状态，防止重复提交。允许将昵称清空为空字符串。

📅每日签到

用户每日可签到领取竞猜币，作为参与竞猜的基础资金来源：

点击「签到 +100」
→
后端悲观锁校验
→
coins += 100
→
写入 coin_log
→
刷新钱包余额

🪙 签到奖励

每次签到获得 +100 竞猜币
常量：DAILY_CHECK_IN_COINS = 100

⏰ 签到规则

按 Asia/Shanghai 自然日限制
每日仅可签到一次
幂等：重复签到返回「今日已签到」

⚠️ 并发安全

后端使用悲观写锁 SELECT ... FOR UPDATE + 事务保证同一用户不会重复签到。通过 lastCheckInYmd 字段（YYYY-MM-DD 格式）判断今日是否已签到。

页面布局解析

  • 导航栏 — 标题「我的竞猜」，左上角返回按钮（青色 ‹ 返回）
  • 段位卡 — 居中大号段位图标（如 🥇）+ 段位名称（如「黄金预言家」）+ 当前门槛文案
  • 数据卡 — 三行 key-value 布局：竞猜币余额、累计竞猜派奖、已结算局数
  • 脚注 — 灰色小字，说明「累计竞猜派奖」仅统计猜中项，平局退还不计入段位

💡 数据来源

接口 GET /api/betting/my-summary
段位由后端 betting-tier.util.ts 按 totalBettingWinnings 自动匹配，前端只做展示。

📜过往竞猜 · betting-history

最近 50 条竞猜记录，每条为一个卡片，包含多层信息：

  • 头行 — 时间（M/D HH:MM）+ 净收益（正数绿色带 +，负数红色）
  • 赛事行 — 「第 X 局 · 疾速冰球 · 赛事名」
  • 对阵行 — 🟥选手名 5 : 3 选手名🟦（含比分）
  • 汇总行 — 本金 X · 发放 Y · 猜中派奖累计 Z
  • 明细行 — 按竞猜类型逐项展示猜中/未中/退还

颜色编码

每条明细根据结果使用不同的视觉标识：

猜中 hit 绿色背景 + 派奖金额
未中 miss 红色背景 + 输掉金额
退还 draw 橙色背景 + 退还金额

🏆竞猜排行 · betting-rank

全服 Top 50 排行榜，按「累计竞猜派奖」降序排列：

  • 排名 — 序号，前 3 名金色高亮
  • 段位图标 — 如 🥇 💎 👑，直观展示段位
  • 用户名 — 超长自动省略
  • 段位名 — 如「黄金预言家」
  • 派奖总额 — 累计竞猜派奖金额
  • 币余额 — 当前竞猜币持有量

💡 排序与刷新

排序规则：totalBettingWinnings DESC → coins DESC → id ASC
仅查询 status=ACTIVE 的用户。每次页面可见（onShow）时重新加载数据。

🎮过往战绩 · JoinMatchHistoryModal

以弹窗形式展示最近 50 条参赛记录，每条包含：

  • 时间 + 结果 — YYYY-MM-DD HH:MM + 「红方胜 / 蓝方胜 / 平局」
  • 玩法 — emoji + 赛事名（🏒 疾速冰球 / 🥊 烈焰拳王 / 🤺 雷霆击剑）
  • 对阵 — 红方 vs 蓝方（缺席显示「机器人」）
  • 比分 — redScore : blueScore
  • 积分变化 — 红蓝双行，显示 ratingBefore → ratingAfter 及差值（+N.NN / -N.NN）

🔗接口清单

个人中心涉及的 8 个后端接口，均需登录鉴权：

接口方法用途
/api/auth/profile GET 获取用户资料（昵称、角色）
/api/auth/profile PUT 更新用户资料（昵称）
/api/betting/wallet GET 竞猜币钱包（余额 + 签到状态）
/api/betting/daily-check-in POST 每日签到
/api/betting/my-summary GET 竞猜汇总（段位 + 派奖 + 局数）
/api/betting/my-history GET 过往竞猜明细（limit=50）
/api/betting/coin-log GET 竞猜币流水（limit=80）
/api/betting/leaderboard GET 竞猜排行榜（take=50）

⚠️异常处理

场景处理方式
钱包加载失败wallet = null，竞猜区不显示余额（不阻塞页面）
签到接口失败Toast 展示错误信息
昵称保存失败Toast 错误信息，弹窗保持打开
子页面加载失败显示错误卡片 + 重试按钮
子页面无数据显示空状态文案（「暂无历史记录」「暂无流水」「暂无排行数据」）
用户资料获取失败自动登出（logout()）

📋局限性与待改进

项目现状建议
管理员视图管理员可看到个人中心，但签到/竞猜对其无意义对 admin 角色隐藏竞猜区或显示管理员专属内容
分页加载所有列表仅做一次全量加载（limit=50/80）数据量大时改为滚动分页加载
下拉刷新页面无下拉刷新支持增加 onPullDownRefresh 支持
头像无头像功能，仅显示昵称可扩展微信头像展示
手机号绑定后端支持 phone 字段但前端未展示增加手机号绑定/换绑功能
积分展示个人中心不直接展示三项玩法积分增加积分概览卡片
竞猜币充值无充值渠道，仅签到获取可考虑扩展充值/购买功能
