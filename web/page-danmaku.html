💬 弹幕：实时互动

所有角色在赛事主页实时发送和查看弹幕消息，为比赛营造氛围、增强参与感

📖页面定位

「弹幕」是嵌入在赛事主页（event.vue）中的实时互动层。所有已登录用户（管理员、选手、观众）在赛事进行过程中均可发送弹幕，消息以飘屏滑动的方式在屏幕上方展示，底部固定输入栏供用户发送新弹幕。

弹幕组件 SessionDanmakuLayer.vue 以场次（Session）为维度隔离消息，切换场次时自动清空并重新拉取。采用HTTP 轮询方式增量获取新消息，无需 WebSocket。

✨功能概览

🎬
飘屏弹幕层
弹幕以半透明白色文字从右向左滑动飘过，模拟经典弹幕效果。3 条泳道轮流分配，互不遮挡。
⌨️
底部输入栏
固定在屏幕底部的输入条，用户可输入文字并发送。支持键盘确认键直接发送（confirm-type="send"）。
🔄
增量轮询
每 2.8 秒向后端拉取增量消息（afterId），首次进入拉取最近 8 条。无需 WebSocket，实现简单稳定。
🛡️
敏感词过滤
服务端校验敏感词，命中时拒绝发送并提示。支持文件词库和环境变量配置，大小写不敏感。
🏷️
按场次隔离
每条弹幕绑定 sessionId，切换场次时清空弹幕层并重新拉取。场次被删除时弹幕级联删除。
👤
显示昵称
弹幕以「昵称：内容」格式展示，发送者身份一目了然。昵称取自用户昵称字段。

📱前端组件详解

弹幕组件 SessionDanmakuLayer.vue 挂载在赛事主页（event.vue）中，包含两个区域：

🎬 飘屏区域（dm-fly-layer）

  • 两种定位模式 — flyAnchorInline=true 时以 absolute 铺满父级赛事模块锚点，仅覆盖 VS/赔率卡区域，不挡底部 Tab；flyAnchorInline=false（默认）时以 fixed 定位在导航栏下方
  • 3 泳道轮询分配 — 每条新弹幕按 laneRound % 3 分配到 0/1/2 泳道，垂直间距 40rpx，避免重叠
  • 随机滑速 — 每条弹幕的 animationDuration 为 11 + random(5) 秒，即 11~16 秒不等，营造自然错落感
  • 自动清理 — 每条弹幕 16 秒后自动从数组中移除；数组上限 16 条，超出时裁剪最早的
  • 文字样式 — 白色文字 + 黑色阴影（text-shadow），确保在各种背景上可读；字号 24rpx，不干扰主要赛事信息
  • pointer-events: none — 飘屏层不拦截触摸事件，不影响下方赛事模块的正常交互

⌨️ 底部输入栏（dm-input-bar）

  • fixed 固定底部 — 输入栏始终贴在屏幕底部（z-index: 95），适配安全区（safe-area-inset-bottom）
  • 输入框 — maxlength="80"，placeholder 为「发条弹幕…」，confirm-type="send" 支持键盘确认键发送
  • 发送按钮 — 发送中（sending=true）或输入为空时禁用；发送成功后自动清空输入框
  • 赛博青蓝配色 — 深色半透明背景 + 青色边框 + 渐变发送按钮，与赛事页面整体风格统一

🔄轮询机制详解

弹幕采用 HTTP 短轮询（非 WebSocket），以 afterId 实现增量拉取：

阶段行为拉取数量
首次进入 afterId=0，拉取该场次最近 N 条（按 id DESC 取再反转为时间正序） limit=50，前端只展示最后 8 条（避免刷屏）
增量轮询 每 2.8 秒 请求一次，where id > afterId，按 id ASC limit=50，全部展示
发送成功 POST 成功后立即更新 afterId，自己的弹幕立即飘屏（不等轮询） 1 条
切换场次 重置 afterId=0，清空弹幕数组，重新首次拉取 —
离开页面 onUnmounted 停止轮询计时器，释放资源 —

💡 为什么用轮询而不是 WebSocket？

弹幕消息频率较低（非高并发直播场景），2.8 秒轮询延迟在可接受范围内。HTTP 轮询的优势：实现简单、无需维护长连接、兼容性好（小程序 WebSocket 在后台切回时可能断连需额外处理）。如果未来弹幕量增大，可平滑升级为 WebSocket 方案。

🔌后端接口

弹幕模块位于后端 modules/danmaku，提供两个 RESTful 接口，均需 JWT 认证：

📥 GET /api/danmaku

按场次增量拉取弹幕
参数：sessionId、afterId（默认 0）、limit（默认 50，最大 100）

📤 POST /api/danmaku

发送弹幕
参数：sessionId（int, ≥1）、content（string, 1~80字）

📤 发送弹幕校验链

后端收到 POST 请求后依次执行以下校验，任一不通过即返回错误：

步骤校验内容错误提示
① 空内容校验 content.trim() 为空 → 拒绝 弹幕内容不能为空
② 长度校验 content.length > 80 → 拒绝（DTO 层 @MaxLength(80)，服务层二次校验） 弹幕内容过长（最多 80 字）
③ 敏感词校验 匹配词库中的敏感词（子串匹配，大小写不敏感）→ 拒绝 弹幕内容包含敏感词，请修改后重试
④ 场次存在校验 sessionId 对应的场次不存在 → 拒绝 场次不存在
⑤ 用户存在校验 JWT 中的 userId 对应的用户不存在 → 拒绝 用户不存在

📤 响应结构

DanmakuMessagePayload

id: number — 弹幕唯一 ID
sessionId: number — 所属场次 ID
userId: number — 发送者用户 ID
displayName: string — 发送者昵称
content: string — 弹幕正文
createdAt: string — 发送时间（ISO 8601）

DanmakuListPayload（GET 响应）

messages: DanmakuMessagePayload[] — 弹幕列表（时间正序）
maxId: number — 当前列表中最大 ID，客户端用于下次轮询的 afterId

🛡️敏感词过滤系统

后端 SensitiveWordFilterService 在服务启动时自动加载敏感词库，对弹幕内容做子串匹配：

📁
文件词库
通过环境变量 DANMAKU_SENSITIVE_WORDS_FILE 指定词库文件路径，默认使用 config/danmaku-sensitive-words.txt，每行一个词，# 开头为注释行。
🔧
环境变量词库
通过 DANMAKU_SENSITIVE_WORDS 逗号分隔传入敏感词，与文件词库合并去重。
🔍
子串匹配 + 大小写不敏感
对弹幕正文做子串包含检查（非全文精确匹配）。ASCII 字母忽略大小写，中文精确匹配。词库按长度降序排列，优先匹配长词减少误报。
🚫
命中即拒绝
命中敏感词时返回 HTTP 400，提示「弹幕内容包含敏感词，请修改后重试」，弹幕不会被存入数据库。

⚠️ 词库为空时不拦截

如果既未配置文件词库也未配置环境变量词库，服务启动时打印警告：「弹幕敏感词库为空，将不拦截任何词」。此行为是为了方便本地开发调试，生产环境务必配置词库。可通过将 DANMAKU_SENSITIVE_WORDS_FILE 显式设为空字符串来禁用文件词库。

🗄️数据库设计

弹幕存储在 session_danmaku 表中，以场次为维度索引，级联跟随场次和用户删除：

字段类型说明
idINT, AUTO_INCREMENT, PK弹幕唯一 ID，用于增量拉取的游标
session_idINT, NOT NULL, FK关联 game_sessions.id，ON DELETE CASCADE（场次删除时弹幕级联删除）
user_idINT, NOT NULL, FK关联 users.id，ON DELETE CASCADE（用户删除时弹幕级联删除）
contentVARCHAR(120), NOT NULL弹幕正文（前端限制 80 字，数据库预留 120 字符余量）
created_atDATETIME(6), AUTO创建时间，精度到微秒

索引

索引名字段用途
IDX_session_danmaku_session_idsession_id按场次查询弹幕
IDX_session_danmaku_session_createdsession_id, created_at按场次 + 时间排序查询

💡 数据库 VARCHAR(120) vs 前后端 80 字限制

数据库字段为 varchar(120)，DTO 层 @MaxLength(80) 限制输入，Service 层二次校验 content.length > 80。数据库多预留 40 字符是为了未来可能的长度调整，无需立即执行 DDL 变更。

👥角色与权限

弹幕功能的角色权限非常简单——所有已登录用户均可发送和查看弹幕，不区分管理员/选手/观众：

角色查看弹幕发送弹幕管理弹幕
管理员 ✓ 可查看 ✓ 可发送 ✗ 无独立管理（无删除/屏蔽接口）
选手 ✓ 可查看 ✓ 可发送 —
观众 ✓ 可查看 ✓ 可发送 —

💡 为什么所有角色都能发弹幕？

弹幕是轻量级的氛围互动工具，而非核心业务操作（如竞猜下注）。允许所有人参与能最大化现场气氛，与"赛事无门槛互动"的产品定位一致。后端仅校验 JWT 有效性（@UseGuards(JwtAuthGuard)），不检查角色字段。

✅ 发送成功后立即展示

用户发送的弹幕在 POST 成功后立即调用 pushFlying() 飘屏展示，无需等待下一轮轮询（2.8s）。这样用户体验是"发出去立刻看到"，而其他用户会在最近一次轮询中看到。

⚙️关键参数速查

参数值位置说明
弹幕最大长度 80 字 DTO / 前端 前端 maxlength=80，后端 @MaxLength(80)，数据库 varchar(120)
