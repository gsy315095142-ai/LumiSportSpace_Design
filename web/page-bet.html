🎰 竞猜：参与押注

管理员开启竞猜后，观众在竞猜面板选择玩法、下注押注的完整流程

📖页面定位

「参与押注」是赛事从报名中进入竞猜阶段后，观众在「我要竞猜」Tab 中进行下注操作的核心环节。管理员手动开启竞猜，观众根据当前游戏类型选择对应玩法并下注。

竞猜玩法根据游戏类型动态显示：胜负盘适用全部游戏，元素之王仅限冰球大师模式，躺平之王仅限拳击，精准总分和精准分差适用全部游戏。观众根据自身竞猜币余额选择玩法和下注金额，比赛结束后由系统自动结算。

🔄竞猜流程概览

竞猜环节从管理员开启竞猜到系统结算，经历以下步骤：

管理员开启竞猜
→
观众选择玩法下注
→
管理员开赛
→
比赛进行中
→
系统自动结算

💡 开放条件

竞猜功能需要管理员手动点击「开启竞猜」后才会开放。前提条件是红蓝双方至少各有 1 名通过审核的正式选手（redUserId / blueUserId 非空），且任一方预期胜率不超过 80%（START_MATCH_VOID_MAX_WIN_RATE = 0.8）。在此之前，观众只能看到竞猜面板但无法操作。

🚫选手：无法参与竞猜

已报名参赛的选手，在竞猜阶段无法对本场进行任何下注操作。待审核状态的选手理论上也不可竞猜（myPendingSide 非空同样被拦截），不过实际场景中待审选手存在说明双方选手尚未确定，此时竞猜本就无法开启：

🚫 选手视角（无法竞猜）

选手 竞猜阶段时

  • 竞猜面板不可操作 — 已报名的选手无法进入竞猜下注界面，系统给出「参赛者不可竞猜」的拦截提示
  • 参赛者身份标识 — 界面顶部明确显示当前用户为「红方选手」或「蓝方选手」，与观众身份区分
  • 撤销参赛后可竞猜 — 如果选手在比赛开始前撤销参赛，身份恢复为观众后即可正常参与竞猜
  • 不影响其他场次 — 选手仅被禁止对本场竞猜；其他场次（如有）仍可正常以观众身份参与

⚠️ 参赛者竞猜限制（代码层面）

代码中 mySide（正式选手）和 myPendingSide（待审核）任一非空都阻止竞猜。不过实际业务上，待审选手存在意味着双方选手尚未确定，管理员无法开启竞猜，因此理论上不会出现「竞猜已开放但有待审选手试图下注」的场景——此处拦截主要是防御性兜底。

🎰观众：竞猜操作界面

观众在「我要竞猜」Tab 下看到完整的竞猜面板，根据当前游戏类型显示对应的竞猜玩法：

👥 观众视角（竞猜面板）

观众 竞猜下注时

  • 本场对阵模块 — 局数 · 竞猜中 · 游戏类型；红蓝双方选手昵称 + 积分清晰展示
  • 竞猜玩法动态显示 — 胜负盘（全游戏）、元素之王（仅冰球大师）、躺平之王（仅拳击）、精准总分、精准分差
  • 个人竞猜资产 — 昵称 + 当前竞猜币余额，实时变动
  • 下注金额输入 — 最低 10 竞猜币，系统自动校验余额

🎲竞猜玩法详解

竞猜玩法根据游戏类型动态显示，观众可任选一种或多种同时下注，每种玩法的下注相互独立：

⚔️
胜负盘
赔率 1.1 ~ 2.0（动态）
押红方胜 或 蓝方胜。赔率根据双方积分实时计算：强者赔率低、弱者赔率高。平局时退还全部下注金额。
适用：全部游戏
🔥
元素之王
赔率 3 倍（固定）
预测冰球大师模式中哪种元素球数量最多。可选冰 / 火 / 风三种元素之一。若多项并列最大，优先级为 ice > fire > wind。
适用：疾速冰球（大师模式）
🥊
躺平之王
赔率 3 倍（固定）
预测拳击比赛中总倒地次数的精确数值。统计整场比赛双方累计倒地的总次数。
适用：烈焰拳王
🎯
精准总分
赔率 20 倍（固定）
预测红蓝双方总得分的精确数值。难度最高，赔率最丰厚。
适用：全部游戏
📏
精准分差
赔率 15 倍（固定）
预测双方分差的精确数值（|红方得分 − 蓝方得分|）。难度较高，回报丰厚。
适用：全部游戏

📊赔率计算机制

⚔️ 胜负盘：动态赔率

胜负盘的赔率不是固定的，而是根据双方选手的积分（Rating）实时计算：

预期胜率 = myRating / ( myRating + enemyRating )
赔率 = ( 1 / 预期胜率 ) × ( 1 − 10% 手续费 )
最终结果钳位在 1.1 ~ 2.0 之间；开赛瞬间锁定为 lockedOdds
  • 强者低赔、弱者高赔 — 积分越高的选手，预期胜率越高，赔率越低（最低 1.1）；弱势方赔率可高达 2.0
  • 10% 手续费 — 从赔付中抽取 10% 作为平台费用（REF_ODDS_FEE_RATE = 0.1）
  • 赔率钳制 — 下限 1.1（REF_MIN_WIN_ODDS），上限 2.0（REF_MAX_WIN_ODDS）
  • 实时浮动 → 开赛锁定 — 报名/竞猜阶段赔率随积分实时浮动；管理员点击「开赛」瞬间锁定为 lockedRedOdds / lockedBlueOdds

💡 覆盖机制

同一类型（如胜负盘）重复下注会覆盖之前的金额和选项，不是叠加。取消下注则全额退还。不同类型（如胜负盘 + 精准总分）之间互不影响，可各自持有下注。

⚠️ 开赛后不可修改

管理员点击「开赛」后，所有下注立即锁定，无法再修改、取消或新增。

⚠️开赛校验与流局处理

管理员点击「开赛」（adminStartMatch）瞬间，系统进行两项校验。任一不通过则触发流局：

👤
选手不齐
任一方无正式选手（redUserId 或 blueUserId 为空）时，比赛无法开始。所有已下注押注作废并全额退还。
⚡
实力过于悬殊
任一方预期胜率 超过 80% 时，判定为实力悬殊。所有竞猜全额退还。

⚠️ 流局后果与代码细节

满足流局条件时：所有已下注押注作废并全额退还，赔率置为 null，betsVoidedAtMatchStart 设为 true。竞猜区顶部展示通知：「开赛时未满足条件（未齐人或实力悬殊），本场押注已作废，竞猜币已全额退还」。

💰管理员操作的退款连锁规则

管理员的以下操作会触发连锁退款，确保观众竞猜币安全：

操作执行阶段退款规则
返回首页 / 重选玩法报名中（betting）或 比赛中（started）当场内存在观众押注时，先全额退还所有观众押注，再收起当前场次
剔除选手报名中（betting）剔除后 guessingOpen 自动重置为 false，若该场已有押注则全部退还
取消本局竞猜报名中（betting）全额退还所有观众下注，局数不变，状态不变

⚙️关键规则汇总

  • 最低下注 10 币 — 单次最低 BET_MIN_AMOUNT = 10 竞猜币
  • 每日签到 100 币 — 新用户初始 0 币，签到获取初始资金，Asia/Shanghai 自然日限一次
  • 参赛者禁猜 — mySide 或 myPendingSide 任一非空即禁止竞猜
  • 管理员手动开启竞猜 — 竞猜非自动开放，需管理员点击「开启竞猜」按钮
  • 实力悬殊拦截 — 开启竞猜时，若预期胜率 > 80%，系统拒绝；开赛时再检一次，触发流局退款
  • 同一类型覆盖 — 重复下注同一玩法会覆盖之前金额和选项，非叠加；不同类型互相独立
  • 竞猜阶段内自由修改 — 开赛前可随时修改金额、更换选项或取消下注，差额多退少补
  • 开赛锁定赔率 — 管理员开赛瞬间，胜负盘动态赔率锁定为 lockedOdds
  • 开赛流局退款 — 开赛时未齐人或实力悬殊 → 全额退还所有押注，betsVoidedAtMatchStart = true
  • 胜负盘平局退款 — 若比赛结果为平局，胜负盘所有下注全额退还
  • 系统自动结算 — 比赛结束后系统根据比分自动结算，猜中者获得「下注额 × 赔率」，未猜中者扣除全部
