🏆 结束：胜负结算

Server 推送结算数据 → 系统自动赔付竞猜 → 选手成绩归档 → 开启下一轮

📖页面定位

「胜负结算」是赛事进入已结算（settled）状态后的收尾环节。比赛结束后，Server 端自动将比分及附加数据（冰球元素统计 / 拳击倒地次数等）写入场次，小程序端同步展示结算结果，系统自动对竞猜玩法进行赔付——猜中者返还「下注额 × 赔率」、未猜中者扣除下注额、胜负盘平局则原路退款。

结算完成后，选手成绩归档到积分榜，管理员可选择开启下一轮进入新的竞技循环，或重选玩法更换游戏类型。

🔄结算流程概览

从比赛结束到结算完成，经历以下环节：

比赛结束 VR 对战完成
→
Server 推送数据 比分 + 附加数据写入场次
→
系统自动结算 逐项赔付竞猜
→
成绩归档 积分榜更新
→
开启下一轮 或重选玩法

💡 结算是自动的

比赛结束后，Server 端自动将结算数据推送至场次记录并触发竞猜赔付。管理员无需手动录入比分或提交结算，小程序端仅展示由服务端写入的数据供查看。

🛡️管理员：结算数据查看

场次进入已结算状态后，管理员页面展示由 Server 端写入的结算数据（只读），以及后续操作按钮：

🛡️ 管理员 · 结算页面

管理员 结算阶段

  • 流程进度条 — 6 步流程，当前停在第 6 步「已结算」
  • 场次信息卡 — 展示本场玩法、模式、场次名等配置信息（只读）
  • 选手名单卡 — 展示红蓝双方选手昵称（只读）
  • 本场结算数据区块 — 由服务端写入并同步，小程序仅供查看：比分（redScore : blueScore）、附加数据（冰球大师模式显示冰/火/风球数及元素之王结果、拳击显示倒地总和）
  • 「开启下一轮」按钮 — 点击后场次状态重置，进入新一轮的报名 → 竞猜 → 比赛 → 结算循环
  • 「重选玩法」按钮 — 点击后退回玩法配置阶段，可更换游戏类型和模式

💡 数据来源说明

结算数据区块中明确标注"以下由服务端写入并同步，小程序仅供查看"。管理员无法在小程序端编辑或修改任何结算数据。

🤺选手：对阵结果页面

结算完成后，选手在小程序端查看本场的胜负结果，展示双方比分与胜负判定：

🤺 选手 · 对阵结果

选手 结算后

  • 比赛名称与状态 — 顶部展示本场比赛名称（matchName），状态标签显示「已结算」
  • 对阵 VS 展示 — 红蓝双方选手昵称 + 最终得分（蓝方得分 : 红方得分），阵营色区分

  • 积分变动提示 — 胜负结算后，选手的 Rating 积分会根据比赛结果更新（Elo 算法），在界面中展示积分变化
  • 本场游戏附加数据 — 如有（冰球元素统计 / 拳击倒地次数等），一并展示供选手回顾

👥观众：竞猜结算结果页面

结算完成后，观众在「我的竞猜」中查看各项玩法的结算结果，赢得的竞猜币自动到账：

👥 观众 · 竞猜结算

观众 结算后

  • 结算结果总览 — 比赛名称 + 最终比分一览，下方逐项展示各竞猜玩法的结算状态
  • 各项下注结算明细 — 每项竞猜显示：下注选项（红方胜 / 蓝方胜 / 比分值 / 分差值等）、下注金额、开赛时锁定的赔率、结算结果（💰 猜中 / ✗ 未中 / ↩ 退款）
  • 猜中赔付 — 猜中的玩法显示绿色标识，赔付金额 = 下注额 × 锁定赔率；竞猜币余额自动增加
  • 未猜中扣除 — 未猜中的玩法显示红色标识，下注金额已扣除，返还 0
  • 平局退款（仅胜负盘） — 若比赛结果为平局，胜负盘所有下注标记为「退款」，金额原路退还
  • 竞猜币余额与段位更新 — 顶部资产信息条展示结算后最新余额；累计盈利（totalBettingWinnings）自动更新并影响「预言家」段位

📊竞猜玩法 · 结算对照

结算时，系统根据 Server 推送的比赛数据逐项判定每种玩法的结果。以下是各玩法的结算依据与赔付逻辑：

⚔️
胜负盘
结算依据：红方得分 vs 蓝方得分
猜中：下注额 × 锁定赔率
未中：扣除全部下注额
平局：退还全部下注额
赔率在开赛时已锁定，以锁定值为准
🔥
元素之王
结算依据：冰/火/风球统计中数量最多的元素
猜中：下注额 × 3 倍
未中：扣除全部下注额
仅疾速冰球大师模式有此玩法
🥊
躺平之王
结算依据：totalKnockdowns（总倒地次数）
猜中：下注额 × 3 倍
未中：扣除全部下注额
仅烈焰拳王有此玩法；需精确匹配倒地次数
🎯
精准总分
结算依据：红方得分 + 蓝方得分 = 精确总分
猜中：下注额 × 20 倍
未中：扣除全部下注额
所有游戏适用
📏
精准分差
结算依据：|红方得分 − 蓝方得分| = 精确分差
猜中：下注额 × 15 倍
未中：扣除全部下注额
所有游戏适用

💡 结算顺序

系统按胜负盘 → 元素之王 → 躺平之王 → 精准总分 → 精准分差的顺序依次结算，每种玩法独立计算、互不影响。观众可能在部分玩法猜中、部分玩法未中，竞猜币余额为各项结算结果的代数和。

📈成绩归档与积分榜更新

结算完成后，系统自动将本场结果写入选手的竞技档案，更新积分榜排名：

归档项 说明 影响
胜负场次 胜方 +1 胜场，负方 +1 负场；平局双方各 +1 平局 胜率（winRate）更新
Rating 积分 根据 Elo 算法调整：胜方加分、负方减分；变动幅度取决于双方赛前积分差距 积分榜排名变化
历史对局记录 双方对阵记录写入数据库（包含比分、游戏类型、附加数据） 个人战绩页可查历史
竞猜盈利累计 观众的 totalBettingWinnings 更新（猜中盈利 − 未中损失） 预言家段位变化

⚙️关键规则汇总

  • 结算由 Server 自动触发 — 比赛结束后 Server 端自动推送结算数据并完成赔付，无需管理员手动操作
  • 结算数据只读 — 管理员页面展示的比分和附加数据由服务端写入，小程序端不可编辑
  • 赔率以开赛锁定值为准 — 结算时使用的赔率是开赛瞬间锁定的值，而非结算时的实时赔率
  • 各玩法独立结算 — 每种玩法按各自的结算依据独立判定，互不影响；盈亏合并计入竞猜币余额
  • 猜中赔付即时到账 — 猜中者的竞猜币在结算后即时增加，无需手动领取
  • 胜负盘平局全额退款 — 仅胜负盘有此特例；其余玩法不设平局退款
  • 积分归档到积分榜 — 结算后选手的 Rating、胜率等竞技数据同步更新，影响后续比赛的赔率计算
  • 开启下一轮重新循环 — 管理员点击「开启下一轮」后，场次状态重置，进入新一轮的报名 → 竞猜 → 比赛 → 结算循环
  • 重选玩法可更换游戏 — 管理员点击「重选玩法」后退回玩法配置阶段，可更换游戏类型和模式
